# Stage 02 — Offer and Value Stack

Build a Grand Slam Offer using Hormozi's Value Equation. Stack bonuses, choose a guarantee, layer urgency. The output is the "what we're selling" that the script will dramatize in Stage 04.

## Inputs

| Source | File/Location | Section/Scope | Why |

| Run metadata | `../01-avatar-positioning/output/video-meta.md` | Full file | Product, format, channel |
| Hormozi reference | `references/value-equation.md` | Full file | The core formula and 4 levers |
| Hormozi reference | `references/bonus-stacking.md` | Full file | How to stack bonuses without cheapening |

## Process

1. Read the positioning brief, the metadata, and all four references.
2. Apply the Value Equation (`value-equation.md`):
   - Score each of the 4 levers (Dream Outcome, Perceived Likelihood, Time Delay, Effort & Sacrifice) on 1-10 for the current product offer.
   - For each lever scoring under 8, propose 1-3 concrete moves to improve it.
3. Build the bonus stack from `bonus-stacking.md`. Aim for 5-10 bonuses each with a perceived value. Total stack value should be at least 10x the price.
4. Pick a guarantee using `guarantee-types.md`. State it explicitly: trigger condition + remedy.
5. Layer urgency/scarcity from `urgency-scarcity.md`. Use real reasons whenever possible.
6. Run the audit.
7. Save.

## Checkpoints

| After Step | Agent Presents | Human Decides |
|---|---|---|
| 2 | Value Equation scoring + proposed lever moves | Which moves are realistic vs. fantasy |
| 4 | 2-3 candidate guarantees | Which one matches the product's risk tolerance |

## Audit

| Check | Pass Condition |
|---|---|
| Stack value > 10x price | Total perceived value of all bonuses comfortably exceeds the asking price |
| Real guarantee | The trigger condition is testable and the remedy is honored |
| Honest urgency | Any urgency claim has a real reason behind it; nothing fake |
| Avatar match | Every bonus speaks to a pain or dream from Stage 01, not generic "value" |
